### Database

데이터베이스의 정의
 * 통합데이터 (Intergrated Data)
   - 데이터베이스는 여러 곳에 산재되어 있지 않고 한 곳에 있어야 한다.
   - 효율성 증진을 위하여 불가피하게 최소의 중복 또는 통제된 중복을 허용
 * 저장데이터 (Stored Data)
   - 컴퓨터가 접근할 수 있는 저장매체에 저장된 데이터
 * 운영데이터 (Operational Data)
   - 조직의 존재 목적이나 기능을 수행하기 위해 반드시 유지되어야 할 데이터의 집합이다
 * 공용데이터 (Shared Data)
   - 한 조직에 있는 여러 응용 시스템들이 공동으로 소유하고 유지하며 이용하는 공용 데이터이다

데이터베이스의 특성
 * 실시간 접근성 (Real-time Accessibility)
   - 수시적이고 비정형적인 질의에 대하여 실시간 처리로 응답할 수 있다
 * 계속적인 변화 (Continuous Evolution)
   - 새로운 데이터의 삽입, 삭제, 갱신을 통해 현재의 정확한 자료를 유지하면서 변화
 * 동시 공용 (Concurrent Sharing)
   - 다수의 사용자가 동시에 원하는 데이터베이스에 접근하여 이용할 수 있다.
 * 내용에 의한 참조 (Contents Reference)
   - 데이터 참조는 데이터베이스에 저장된 레코드들의 위치나 주소에 의해서가 아니라 사용자가 요구하는
     데이터의 내용, 즉 값에 따라 참조된다
      
데이터베이스 관리 (DBMS)의 특징
 * 데이터 중복 최소화
 * 데이터 공동 이용
 * 데이터 일괄성 유지 (중복을 제어하고 통제를 통해 데이터의 일괄성을 유지)
 * 데이터 무결성 유지 (데이터베이스에 저장된 값과 현실세계의 실제 값이 일치하는 정확성을 의미, 정합성)
 * 데이터 보안 유지 

스키마(Schema)
 1) 정의
   - DB의 구조와 제약 조건에 대한 전반적인 명세를 기술한 것
   - DB를 구성하는 데이터 객체, 설징, 관계, 데이터의 조작 및 이들 데이터 값들이 갖는 제약 조건에 관한 정의를 총칭
   - 메타데이터(Meta-data)라고도 한다

 2) 스키마의 3계층
  * 스키마는 사용자의 관점에 따라 외부, 개념, 내부 스키마로 구분된다
  * DBMS는 외부적 스키마에 따라 명시된 사용자의 요구를 개념적 스키마에 적합한 형태로 변경하고 이를 내부로 변환
  
  - 외부 스키마(External Schema)
    일반 사용자나 응용 프로그래머가 각 개인의 입장에서 필요로 하는 DB의 논리적 구조를 정의한 것
    공용의 의미보다 어느 개인이나 특정 응용에 한정된 논리적인 데이터 구조이다
    전체 DB의 한 논리적인 부분으로 볼 수 있으므로 서브스키마(Sub Schema) 또는 사용자 뷰라고 한다.
    (ex 모델하우스)

  - 개념 스키마(Conceptual Schema)
    DB의 전체적인 논리적 구조로서 모든 응용 프로그램이나 사용자들이 필요로 하는 데이터를 종합한 
    범기관적 입장에서 본 조직 전체의 데이터베이스를 의미한다
    개체 간의 관계와 유지해야 할 제약 조건을 나타낸다
    데이터베이스 파일에 저장되는 데이터의 형태를 나타낸 것으로 단순히 스키마(Schema)라고도 한다
    
  - 내부 스키마 (Internal Schema)
    물리적 저장장치의 입장에서 본 데이터베이스 구조를 정의 한 것이다
    실제로 데이터베이스에 저장될 레코드형식을 정의하고 저장 데이터 항목의 표현 방법, 
    내부 레코드의 물리적 순서등을 나타냄
    저장스키마(Storage Schema)라고도 한다
    
관계 데이터 모델
 1) 릴레이션 (Relation)
   - 행(Row)과 열(Column)로 구성하는 테이블을 관계 데이터 모델에서는 릴레이션이라고 한다
   - 릴레이션 스킴 : 릴레이션의 논리적 구조를 나타내며, 릴레이션 스키마 또는 릴레이션 내포(Intension)
   - 릴레이션 인스턴스 : 릴레이션의 실제 값들을 나타내며, 릴레이션 어커런스(Occurrence)또는 외연(Extention)

 2) 튜플 (Tuple)
   - 릴레이션을 구성하는 각각의 행
   - 속성값들의 집합으로 구성
   - 파일구조의 레코드와 같은 의미
 
 3) 속성 (Attribute)
   - 릴레이션을 구성하는 각각의 열
   - 개체의 특성이나 상태
   - 데이터베이스를 구성하는 가장 작은 논리적인 단위
   - 속성의 수를 차수(Degree)라고 한다

 4) 도메인 (Domain)
   - 하나의 속성이 취할 수 있는 같은 타입의 원자값 (Atomic Value)들의 집합
   - 정의된 속성은 반드시 해당 도메인 내에서만 값을 취할 수 있다.
     (학생 릴레이션으 학년 속성의 도메인은 1~4이다)
     
 5) ER스키마의 관계 스키마 사상(Mapping Rule)
   - 1:1 관계
     각 개체를 릴레이션으로 만든 후 둘 중 한 개체에 외래키로 상대방을 포함시키고 관계를 포함한다
     (교수 - 1 - 강의 - 1 - 과목)
   
   - 1:N 관계
     각 개체를 릴레이션으로 만든 후 1의 기본키를 N에 외래키로 포함시키고 관계 역시 N에 포함시킨다
     (학과 - 1 - 소속 - N - 학생)
     
   - M:N 관계
     모든 개체와 관계를 독립된 릴레이션으로 만든 후, 관계 릴레이션에는 각 개체의 기본키를 외래키로 포함시킨다
     (학생 - N - 수강 - M - 과목)

키
  튜플을 유일하게 식별할 수 있는 속성 집합을 그 릴레이션의 키라고 한다.
  튜플을 검색하거나 정렬할 때 튜플들을 서로 구분할 수 있는 기준이 되는 속성이다.
  
  * 키의 특성
  유일성 : 하나의 키 값으로 하나의 튜플만을 유일하게 식별할 수 있다
  최소성 : 키를 구성하는 속성 하나만 제외시켜도 유일성이 깨지도록 꼭 필요한 최소의 속성으로 구성된다
  
  1) 슈퍼키 (Super Key)
    - 튜플을 유일하게 구분하기 위해 한 개 이상의 속성들의 집합으로 이루어진 키
    - 한 릴레이션에서 유일성은 만족시키지만, 최소성은 만족시키지 못한다

  2) 후보키 (Candidate Key)
    - 튜플을 유일하게 구분할 수 있는 최소 슈퍼키
    - 한 릴레이션에서 유일성과 최소성을 모두 만족시킨다

  3) 기본키 (Primary Key)
    - 후보키 중에서 대표로 지정된 키
    - 중복값이나 널 값을 가질 수 없다
    - 주키, 주 식별자라고 한다.

  4) 대체키 (Alternate Key)
    - 하나의 릴레이션에 존재하는 후보키들 중에서 기본키를 제외한 나머지 후보키들을 말한다.
    - 후보키 - 기본키 = 대체키

  5) 외래키 (Foreign Key)
    - 어떤 릴레이션 R1의 기본키의 값들과 일치함을 요구하는 다른 릴레이션의 R2의 한 속성을 말한다
    - 외래키로 지정되면 참조 릴레이션의 기본키에 없는 값은 입력할 수 없다
    - 참조 무결성 제약 조건과 밀접한 관계를 가진다
