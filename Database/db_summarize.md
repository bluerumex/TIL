데이터베이스의 정의
 * 통합데이터 (Intergrated Data)
 	- 데이터베이스는 여러 곳에 산재되어 있지 않고 한 곳에 있어야 한다.
 	- 효율성 증진을 위하여 불가피하게 최소의 중복 또는 통제된 중복을 허용
 * 저장데이터 (Stored Data)
 	- 컴퓨터가 접근할 수 있는 저장매체에 저장된 데이터
 * 운영데이터 (Operational Data)
 	- 조직의 존재 목적이나 기능을 수행하기 위해 반드시 유지되어야 할 데이터의 집합이다
 * 공용데이터 (Shared Data)
 	- 한 조직에 있는 여러 응용 시스템들이 공동으로 소유하고 유지하며 이용하는 공용 데이터이다

데이터베이스의 특성
 * 실시간 접근성 (Real-time Accessibility)
 	- 수시적이고 비정형적인 질의에 대하여 실시간 처리로 응답할 수 있다
 * 계속적인 변화 (Continuous Evolution)
 	- 새로운 데이터의 삽입, 삭제, 갱신을 통해 현재의 정확한 자료를 유지하면서 변화
 * 동시 공용 (Concurrent Sharing)
 	- 다수의 사용자가 동시에 원하는 데이터베이스에 접근하여 이용할 수 있다.
 * 내용에 의한 참조 (Contents Reference)
 	- 데이터 참조는 데이터베이스에 저장된 레코드들의 위치나 주소에 의해서가 아니라 사용자가 요구하는
 	  데이터의 내용, 즉 값에 따라 참조된다
      
데이터베이스 관리 (DBMS)의 특징
 * 데이터 중복 최소화
 * 데이터 공동 이용
 * 데이터 일괄성 유지 (중복을 제어하고 통제를 통해 데이터의 일괄성을 유지)
 * 데이터 무결성 유지 (데이터베이스에 저장된 값과 현실세계의 실제 값이 일치하는 정확성을 의미, 정합성)
 * 데이터 보안 유지 

스키마(Schema)
